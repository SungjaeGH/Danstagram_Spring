# Danstagram_Spring

## Commit Convention
### Type
* feat: 기능 추가, 삭제, 변경
* fix: 버그 수정
* refactor: 코드 리팩토링
* style: 코드 형식, 정렬 등의 변경. 동작에 영향 x
* test: 테스트 코드 추가, 삭제 변경
* docs: 문서 추가 삭제 변경. 코드 수정 x
* etc: 위에 해당하지 않는 모든 변경

### Description
* 한 줄당 72자 이내로 작성
* 최대한 상세히 작성(why - what)
